# סיכום שינויים - גרף תחזית תזרים ותיקון לולאת סנכרון (07/06/2026)

בסשן זה טיפלנו בשני נושאים נפרדים:

## 1. Tooltip מותאם RTL לגרף התחזית בעמוד `/budget`
- **הבעיה**: בגרף "תזרים 12 חודשים קדימה" הוצג tooltip ברירת מחדל של הדפדפן דרך `title`. זה לא נתן שליטה על יישור מימין לשמאל, לא היה נוח לקריאה ב-RTL, ולא איפשר עיצוב עקבי עם שאר הממשק.
- **הפתרון**:
  - עדכנתי את `components/budget/CashflowForecast.tsx`.
  - החלפתי את `title` ב-tooltip מותאם לכל טיק/עמודת חודש.
  - ה-tooltip מוצג ב-hover וגם ב-focus, עם `dir="rtl"`, יישור טקסט לימין, ותוכן מובנה:
    - שם החודש
    - סכום התזרים
    - סטטוס התזרים
    - אירועים מיוחדים אם קיימים
  - הוספתי גם `aria-label` כדי לשמור על נגישות למקלדת ולקוראי מסך.

## 2. תיקון `Maximum call stack size exceeded` בסנכרון onboarding
- **הבעיה**: נוצרה לולאת קריאה סינכרונית בין `syncOnboardingToStores()` לבין ה-handler בעמוד `/budget`. הפונקציה הפיצה אירועי עדכון, וה-`handler` חזר וקרא לה שוב, עד קריסת stack.
- **הפתרון**:
  - עדכנתי את `lib/onboarding-sync.ts`.
  - הוספתי guard ברמת המודול (`isSyncingOnboarding`) שמונע re-entry בזמן שהסנכרון כבר רץ.
  - עטפתי את גוף `syncOnboardingToStores()` ב-`try/finally` כדי לוודא שה-flag משתחרר גם אם אחת מפעולות הסנכרון נכשלת.
  - כך נשמרים עדכוני הסנכרון הרגילים, אבל נמנעת רקורסיה אינסופית.

## אימות
- הרצת `npm run typecheck -- --pretty false` עברה בהצלחה לאחר השינויים.
